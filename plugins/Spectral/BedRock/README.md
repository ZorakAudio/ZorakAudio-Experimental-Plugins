# BedRock

## What it is
BedRock is an **adaptive pitch-aware subharmonic engine**.

It does not behave like a simple bass boost. The current source tracks pitch, generates controlled lower-octave content, adds optional transient micro-bursts, and manages low-end budget so the result stays weighty without instantly turning into swamp.

---

## Why use it
Use it when a source needs:

- more foundation
- controlled sub extension
- pitch-aware low-end weight
- impact reinforcement
- translated bass that still reads on smaller playback

---

## Quick start
1. Choose a **Style** that matches the job: Anchor, Cinematic, or Impact.
2. Raise **Amount** for the overall generated foundation.
3. Use **Depth (-2)** to decide how strongly the deeper octave is emphasized.
4. Use **Tightness** to keep the sub under control.
5. Set **Tone** for darker/deeper versus brighter/more translated low-end.

---

## Main controls
### Style
Selects the internal behavior profile. Anchor is steadier, Cinematic blooms more, and Impact hits harder.

### Amount
Overall amount of generated foundation.

### Depth (-2)
How much the deeper octave weighting is emphasized.

### Tightness
How tightly gated, confidence-limited, and budget-controlled the low layer behaves.

### Tone
Moves the generated layer darker/deeper or brighter/more translated.

---

## Routing / notes
Normal use: insert it directly on the bass, kick, impact, or low source.

Optional use: feed a sidechain trigger to **channels 3/4**. In the current source, that lets the gate follow the sidechain while the pitch still follows the source.

---

## In one sentence
BedRock synthesizes controlled, pitch-aware low-end foundation instead of merely boosting whatever bass is already there.
